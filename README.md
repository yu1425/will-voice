# うぃる AIボイス 🎾

社会人テニスサークル **WILL.tennis** のマスコットキャラクター「**うぃる**」に、
音声で話しかけると AIっぽく返答してくれる Web アプリの **無料MVP** です。

- 🎤 マイクで話しかける（ブラウザ標準の音声認識）
- 💬 認識テキストとうぃるの返答をチャット形式で表示
- 🔊 うぃるの返答を読み上げ（録音音声 / ブラウザ標準音声 / VOICEVOX:ずんだもん）
- 🎾 **進行モード** — テニス会の流れに沿ってうぃるが進行アナウンスを読み上げ
- 📱 スマホで使いやすい LINE 風のやさしい UI
- 🏠 **PWA 対応** — iPhone のホーム画面にアイコンを追加してアプリのように使える

> **このMVPは外部AI APIを一切使いません。**
> 返答はキーワードに応じた「モック返答」、音声はブラウザ標準機能を使うので、
> **APIキー不要・完全無料**で動きます。将来 Claude / OpenAI / TTS API に差し替えやすい設計です。

## iPhoneのホーム画面に追加する（PWA）

iPhoneの Safari でアクセスし、ホーム画面に追加するとアプリのように起動できます。

### 追加手順

1. iPhone の **Safari** で `https://will-voice.vercel.app/` を開く
2. 画面下部の **共有ボタン**（□↑）をタップ
3. 「**ホーム画面に追加**」を選ぶ
4. 名前が「**うぃるくん進行**」になっていることを確認
5. 右上の「**追加**」をタップ

追加後はホーム画面のアイコンからアプリのように起動できます。
URLバーやブラウザのタブバーが表示されず、全画面でアプリっぽく動きます。

> **Chrome では「ホーム画面に追加」が iOS ではうまく動かない場合があります。Safari をお使いください。**

### PWA の構成

| 項目 | 設定値 |
| --- | --- |
| アプリ名 | うぃるくん進行 |
| ショートネーム | うぃる |
| テーマカラー | グリーン (#5fae6e) |
| 背景色 | 淡いグリーン (#F3FAF4) |
| 表示モード | standalone（全画面アプリ風） |
| アイコン | うぃるくんアイコン 192px / 512px |

元音声ファイルのバックアップは `public/audio/うぃる音声/` に保持しています。

## VOICEVOX（ずんだもん）で読み上げる

うぃるの返答を **VOICEVOX のずんだもん** で読み上げられます（無料・APIキー不要）。
VOICEVOX が起動していないときは、自動的にブラウザ標準音声にフォールバックします。

### 1. VOICEVOX をインストール・起動

1. 公式サイトから VOICEVOX をダウンロード: <https://voicevox.hiroshiba.jp/>
2. アプリを起動すると、ローカルで VOICEVOX ENGINE が `http://127.0.0.1:50021` で待ち受けます
3. アプリは起動したままにしてください（このアプリは ENGINE に接続します）

### 2. `.env.local` を作成

```bash
cp .env.example .env.local
```

`.env.local` の以下の値を確認してください。

```
VOICEVOX_ENGINE_URL=http://127.0.0.1:50021
VOICEVOX_DEFAULT_SPEAKER_ID=3
```

### 3. ずんだもんの speaker id を確認する

`speaker id` は VOICEVOX のバージョンで変わる可能性があります。
次のいずれかで「ずんだもん」の `styles[].id` を確認してください。

- アプリ起動中に <http://localhost:3000/api/voicevox/speakers> を開く
- もしくは <http://127.0.0.1:50021/speakers> を直接開く

JSONの中で `"name": "ずんだもん"` の項目を探し、好きなスタイル（ノーマル / あまあま 等）の `id` を控えて、`.env.local` の `VOICEVOX_DEFAULT_SPEAKER_ID` に設定してください。

### 4. アプリで切り替える

画面上部の「読み上げ: 標準音声 / 録音音声 / **VOICEVOX**」ボタンで切り替えられます。
「VOICEVOX」を選ぶと、その下に詳細パネルが表示されます。

- **音声: VOICEVOX:ずんだもん** — クレジット表記
- **接続状態ピル** — 「接続済み / 未接続 / 標準音声にフォールバック中 / 確認中…」
- **話者スタイル選択** — ENGINE から取得した「ずんだもん」のスタイル一覧（ノーマル / あまあま / セクシー / ツンツン など）から選べます
- **↻ ボタン** — 話者一覧を再取得（VOICEVOXを後から起動した場合に使えます）

選んだスタイルは `localStorage` に保存され、次回起動時に復元されます。

VOICEVOX を選んだときに ENGINE が見つからなかった場合は、

> VOICEVOXが見つからなかったため、標準音声で読み上げます

と表示し、ブラウザ標準音声に自動でフォールバックします（接続状態は「標準音声にフォールバック中」と表示されます）。

### 話者スタイルの初期値について

- 起動時、画面の話者スタイル選択は次の優先度で初期化されます:
  1. `localStorage` に保存された前回選択のスタイル id
  2. `.env.local` の `VOICEVOX_DEFAULT_SPEAKER_ID`
  3. ずんだもんの最初のスタイル
- **speaker id は VOICEVOX のバージョン/環境で変わる可能性があります。** 接続後の話者選択UIで実際に取得した一覧から選び直してください。

### 音声チューニング

VOICEVOX パネルの「**音声チューニング**」から、プリセットを選んでその場で音声を調整できます。
選んだプリセットは `localStorage` に保存され、次回起動時に復元されます。

| プリセット | speed | pitch | intonation | postPhoneme | 用途 |
| --- | --- | --- | --- | --- | --- |
| 標準 | 1.0 | 0.0 | 1.0 | 0.2 | シンプルな読み上げ |
| 明るめ（既定） | 1.05 | 0.0 | 1.15 | 0.2 | うぃるらしく少し明るく |
| 聞き取りやすさ重視 | 0.95 | 0.0 | 1.05 | 0.25 | 屋外・ノイズ環境で明瞭に |
| ゆっくり | 0.88 | 0.0 | 1.0 | 0.3 | 初参加者が多い日・説明重視 |

> **コート推奨設定**: 屋外のテニスコートでは「**聞き取りやすさ重視**」または「**ゆっくり**」がおすすめです。
> スピーカーの音量が大きい場合は「標準」でも十分に聞き取れます。

「**この設定で音声テスト**」ボタンを押すと、選択中のプリセットで次のセリフを読み上げます:

> 本日はご参加ありがとうございます。ショートラリーとボレーボレーを、それぞれ5分ずつ行います。聞こえ方に問題がなければ、この設定で進行してください。

開始前にスピーカーや接続を確かめるのに使ってください。

パラメータの詳細を直接変えたい場合は `app/api/voicevox/tts/route.ts` の `DEFAULT_AUDIO_PARAMS`（サーバー側フォールバック値）か、`app/page.tsx` の `AUDIO_PRESETS` を編集してください。

### 読み上げテキスト補正（読み辞書）

VOICEVOX もブラウザ標準音声も、固有名詞や英字を誤って読むことがあります。
本アプリでは読み上げ直前に **`sanitizeForVoicevox()`**（`lib/voicevoxText.ts`）を通して自動補正します。

**表示テキストやコピーテキストは一切変えません。** 変わるのは読み上げに送る文字列だけです。

#### どこで補正がかかるか

読み辞書は **標準音声・VOICEVOX の両モード**で効きます。

| モード | 補正の適用箇所 |
| --- | --- |
| 標準音声 | `lib/speech.ts` の `speakText()` |
| VOICEVOX | `lib/voicevox.ts`（ENGINE送信前）＋ `app/api/voicevox/tts/route.ts`（サーバー側） |

VOICEVOX はクライアント側とサーバー側の両方で補正を通しますが、**何度適用しても結果は変わらない（冪等）**ように作ってあるため安全です。クライアント側でも補正するので、`npm run dev` のサーバーが古いコードのままでも正しい読みが届きます。

> 💡 補正が効かないと感じたら、まず音声モードがどちらでも構いません（両方に効きます）。それでも変わらない場合はブラウザを再読み込み（ハードリロード）してください。

| `4ポイント先取` | よんポイントせんしゅ |

| `斜め方向` | ななめ方向 |
| `揃って` | そろって |
| `強打` / `先頭` / `先取` / `交代` / `集合` など | きょうだ / せんとう / せんしゅ / こうたい / しゅうごう など（テニス進行語句） |

#### 補正例

```
入力: WILL.tennisでは、LINEノートにスクショで報告をお願いします🎾
出力: ウィルテニスでは、ラインノートにスクリーンショットで報告をお願いします。

入力: 次はサーブリターンです。4ポイント先取、デュースなしです。
出力: 次はサーブ、リターンです。よんポイントせんしゅ、デュースなしです。
```

#### 「方」の読みについて

「方」は文脈によって「かた（人を指す）」と「ほう（方向・どちら）」の 2 通りの読みがあります。
`sanitizeForVoicevox()` では **フレーズ単位でのみ** `かた` 読みに変換し、`片方` / `両方` / `方向` / `方針` / `方法` などが壊れないようにしています。

| 変換するフレーズ例 | 変換後 |
| --- | --- |
| `初めての方` | はじめてのかた |
| `初参加の方` | はじめて参加されるかた |
| `遅れて来る方` / `遅れてくる方` | おくれてくるかた |
| `遅れて来られる方` | おくれてこられるかた |
| `待っている方` / `打っている方` / `休んでいる方` | まっているかた など |
| `来る方` / `来られる方` / `いる方` / `いらっしゃる方` | くるかた / こられるかた / いるかた / いらっしゃるかた |
| `次に入る方` / `先に入る方` / `後から入る方` | つぎに入るかた など（「入る」は漢字のまま） |
| `近くの方` / `隣の方` / `相手の方` / `ペアの方` | ちかくのかた など |
| `参加の方` / `参加する方` / `参加される方` | 参加のかた など（「参加」は漢字のまま） |
| `久しぶりの方` / `体調が悪い方` / `不安な方` など | （同様にかた読み） |
| `分からない方は` / `わからない方は` | わからないかたは（「〜方」の短い形より前に配置） |

変換しないもの（意図的に除外）: `片方` / `両方` / `方向` / `方針` / `方法`
（`打ちやすい方` のように「ほう（どちら）」の意味で使う `方` も辞書に入れていません）

> 💡 **設計方針**: 直すのは「方（ほう→かた）」の読みだけで、`参加` `入る` などの漢字はそのまま残します（VOICEVOX が正しく読むため）。すべてをひらがなにすると、かえって不自然になりやすいためです。

#### 「初参加」を「しょさんか」と読ませない

`初参加` を `しょさんか` と読むのは不自然なので、自然な言い換えに変換します。
「初参加の方」は方フレーズ側で先に処理され、残りの形を後段で変換します（素の `初参加` は必ず最後）。

| 元の表記 | 変換後 |
| --- | --- |
| `初参加の方` | はじめて参加されるかた |
| `初参加者` | はじめて参加される人 |
| `初参加です` | はじめての参加です |
| `初参加` | はじめて参加 |

#### テニス進行語句について

テニス会の進行で頻出する語句も読み上げ辞書で管理しています（`lib/voicevoxText.ts` の「テニス進行の語句」セクション）。
VOICEVOX が誤読しやすい単語や、送り仮名で読み方が変わる単語を中心にカバーしています。

#### 辞書の追加方法

読み方がおかしい語を見つけたら、`lib/voicevoxText.ts` の `REPLACEMENTS` 配列に追加してください。
**長いフレーズを先に置換するため、長いものを配列の前の方に追加してください。**
「方」フレーズを追加する場合は「人を指す「方」のフレーズ」セクションに、長さ順で追加してください。

```ts
// 例: 「苦手な方」を追加する場合
[/苦手な方/g, "にがてなかた"],
```

#### voiceText 自体も読みやすく（辞書任せにしない）

重要なセリフは、辞書だけに頼らず `lib/tennisFlowScripts.ts` の `voiceText` 自体も自然に読めるよう調整しています。
**表示用の `displayText` は変えず**、読み上げ用の `voiceText` だけを直します。

- **進行セリフは実際の運営原稿をベースに差し替えています。**
  表示テキスト（`displayText`）は原稿に近い自然な日本語、読み上げテキスト（`voiceText`）はVOICEVOXで自然に聞こえるよう調整済みです。
- **`voiceText` の「方」は人を指す場合は `かた`（ひらがな）で書きます。**
  辞書での変換にも頼りますが、voiceText 側に直接書くことで確実に読みが制御できます。
- **`テニス歴が浅い方` の表現は使いません。**
  VOICEVOXで不自然になりやすいため、`初心者のかた` / `久しぶりのかた` に言い換えています。
- **`表の行と列` の表現は使いません。**
  乱数表の説明では「上から順番に進める」「ペアと対戦相手を確認する」など自然な表現に置き換えています。
- **`ボレーボレー` はそのまま読み上げます。**
  以前は `ボレー、ボレー` に変換していましたが、イントネーションが不自然になるため廃止しました。

```
displayText（表示）: 遅れて来るかたもいるので、全体の自己紹介は人数が揃ってきた…
voiceText（読み上げ）: …人数がそろってきた…   ← 「かた」に直し、「そろって」、読点を追加
最終（VOICEVOXへ）  : …人数がそろってきたタイミングで行います。
```

#### 読み上げ確認（デバッグ用）

進行モードの **準備モード** のステップカード下部に、折りたたみの「**読み上げ確認**」があります（通常は閉じています）。開くと次の 3 つを確認できます。

1. **表示用テキスト** — 画面に出している `displayText`（編集中なら編集後）
2. **読み上げ用テキスト** — 実際に読み上げに渡す文（長さ設定・人数調整プレフィックス込み・補正前）
3. **補正後にVOICEVOXへ送る最終テキスト** — `sanitizeForVoicevox()` 適用後の最終形

読みがおかしいときは、まずここで「最終テキスト」を見て、辞書（`lib/voicevoxText.ts`）か voiceText のどちらを直すか判断できます。

#### デバッグ用ログ

開発環境（`NODE_ENV=development`）では、補正前後のテキストがサーバーのターミナルに出力されます。

```
[VOICEVOX sanitize] before: 乱数表で4ポイント先取、初参加の方もいます
[VOICEVOX sanitize] after:  らんすうひょうでよんポイントせんしゅ、はじめて参加されるかたもいます。
```

公式ガイドライン: <https://zunko.jp/con_ongen_kiyaku.html> / <https://voicevox.hiroshiba.jp/term/>

### Vercel など公開環境にデプロイする場合の注意

このアプリの `/api/voicevox/*` は **サーバー側（Next.jsが動くマシン）から** VOICEVOX ENGINE に接続します。
そのため、Vercel など外部にデプロイした場合、**Vercel のサーバーはあなたのローカルPCの VOICEVOX には繋がりません。**

- ローカルで `npm run dev` して使う → ✅ 動きます
- Vercel に公開して、自分のPCの VOICEVOX に繋ぐ → ❌ 繋がりません
- 公開でも VOICEVOX を使いたい場合は、**VOICEVOX ENGINE を別途サーバー（VPS等）で起動**し、`VOICEVOX_ENGINE_URL` をそのアドレスに設定する構成が必要です

VOICEVOX が利用できない環境では、ブラウザ標準音声で問題なく動作します。

## テニス会 進行モード 🎾

WILL.tennis の開催中、管理者がボタン1つで、うぃるに**進行アナウンスを読み上げてもらえる**モードです。
画面上部のタブで「通常チャット」「進行モード」を切り替えられます。

### モード切替

画面上部のタブをタップ:

- **通常チャット** — マイクで話しかけてうぃると会話
- **進行モード** — 進行アナウンスを表示・読み上げ

選んだタブは `localStorage` に保存され、次回起動時にも復元されます。

### 進行ステップ

進行モードでは、以下の10ステップが用意されています。

1. 開始あいさつ
2. ショートラリー・ボレーボレー
3. ロングラリー
4. クロスラリー
5. サーブリターン
6. ゲーム前の自己紹介
7. 乱数表の使い方
8. 試合ルール・配慮事項
9. ミニゲーム
10. 終了あいさつ

ステップは「← 前へ / 次へ →」ボタンか、ステップ一覧チップから直接ジャンプできます。
現在のステップは `localStorage` に保存され、再開時に復元されます。

### 準備モードと当日モード

進行モードの中には、さらに **「準備モード」** と **「当日モード」** の2つの表示モードがあります。タブで切り替えられ、選択は `localStorage` に保存されます。

| | 準備モード | 当日モード |
| --- | --- | --- |
| 想定シーン | 開催前にセッティング | コート上でスマホ片手 |
| 今日の条件入力 | ◯ | (準備モードに任せる) |
| 進行プラン表示 | ◯ | — |
| セリフ編集 | ◯ | — |
| セリフ長さ切替 | ◯ | (準備モードの設定が反映) |
| 初心者向け補足 | ◯ | — |
| 通常の注意喚起 | ◯ | — |
| 大きい読み上げ/停止/次へボタン | — | ◯ |
| クイック注意喚起 | — | ◯ |
| 音声テスト | — | ◯ |
| タイマー | ◯ | ◯ |

ベースとなる条件（コート数・参加人数・雰囲気など）と、セリフの編集・長さは **準備モード** で済ませておけば、**当日モード** はその設定をそのまま読み上げます。

### 当日モードの使い方

当日、テニスコート上では **「当日モード」** に切り替えます。

- **大きい▶読み上げボタン**: 現在のステップを読み上げ
- **大きい■停止ボタン**: 読み上げ中はそのまま「停止」に変化
- **大きい「次へ →」ボタン**: 次のステップへ進む（次のステップ名も併記）
- **小さい「← 戻る」ボタン**: ひとつ前へ
- **現在のステップ**: 画面上部に大きく表示。次のステップ名も小さく予告表示
- **タイマー**: 5分／10分／一時停止／リセット
- **クイック注意喚起**: 6つの短いセリフをワンタップで読み上げ
- **🔊 音声テスト**: 開始前の音量確認用

屋外での視認性のため、現在ステップは緑のヒーロー領域に白文字で大きく表示しています。

### 音声テストボタン

当日モード下部の **「🔊 音声テスト」** ボタンを押すと、次のセリフをうぃるが読み上げます。

> お疲れ様です。
> うぃるの音声テストです。

開始前にスピーカーや VOICEVOX 接続を確かめるのに使ってください。

### クイック注意喚起ボタン

当日モードの「クイック注意喚起」グリッドには、コート上で即使える短いお願いが並びます。

- 交代お願いします
- 強打は控えめに
- 体調無理しないで
- 初参加の方にも声かけ
- 次のメニューに移ります
- 集合お願いします

セリフは [`lib/flowCautions.ts`](lib/flowCautions.ts) の `FLOW_QUICK_CAUTIONS` で編集できます。

### 進行状況の保存

現在のステップ・表示モード（準備/当日）・タイマー設定値・条件入力・編集テキストはすべて `localStorage` に保存され、次回起動時にそのまま復元されます。途中でアプリを閉じても、当日の進行を続けられます。

### 今日の進行プラン作成

進行モードを開くと、一番上に **「今日の進行プラン」** パネルがあります。次の条件を入力すると、その場で時間配分プランを自動で組み立てて表示します（AI APIは使わない、ルールベース）。

| 入力項目 | 内容 |
| --- | --- |
| 参加人数 | 数値（0〜40） |
| コート数 | 1面 / 2面 |
| 開催時間 | 1時間 / 1.5時間 / 2時間 / 3時間 |
| 初参加者の人数 | 数値（0〜40） |
| 初心者が多い | チェックボックス |
| 雰囲気 | ゆるめ / 標準 / しっかり |

プランは「`0:00〜0:10 開始あいさつ・ショートラリー`」のような形式で表示され、**プランをコピー** ボタンで LINE にそのまま貼れます。

入力に応じて、ロング/クロス/サーブの時間を少し伸ばしたり、ミニゲームを長くしたり、自己紹介の時間を増やしたりと、軽い微調整が入ります。

### コート数切替（1面 / 2面）

進行プランパネル内の **コート数** で切り替えられます。
**クロスラリー** / **サーブリターン** / **乱数表の使い方** は、コート数に応じて文面が自動で切り替わります（移動指示やコート振り分けの内容が変わります）。

### 参加人数に応じた文面調整

> ⚠️ **現在、人数プレフィックスは無効化しています。**
> 10人前後では「人数多めなので」とアナウンスする必要がないと判断したため、
> `getCrowdPrefix()` は常に `null` を返すよう設定しています（`lib/tennisFlowScripts.ts`）。
> 必要になった場合は同関数の実装を復活させてください。

~~進行プランで参加人数を入力していると、対象ステップでは人数に応じた一言が冒頭に自動で加わります。~~

### セリフの長さ切替

各ステップカードに **「読み上げの長さ: 通常 / 短め / かなり短め」** トグルがあります。

- **通常**: 全体を丁寧に読み上げ
- **短め**: 要点だけを短めに
- **かなり短め**: 一言で済ます

選択は `localStorage` に保存され、次回起動時にも復元されます。

### 初心者向け補足ボタン

各ステップカードの **「初心者向け補足を読む」** ボタンを押すと、そのステップ用の短いやさしい補足をうぃるが読み上げます。

例（ロングラリー）:
> 強く打つより、まずは相手が返しやすいボールでつなぐ意識でお願いします。
> ミスしても大丈夫なので、声をかけ合いながら楽しく進めましょう。

補足は [`lib/tennisFlowScripts.ts`](lib/tennisFlowScripts.ts) の各ステップの `beginnerTip` に書かれています。

### やさしい注意喚起ボタン

進行モードの **「やさしい注意喚起」** パネルを開くと、6つのボタンが並びます。押すと、うぃるが短くやさしく読み上げます。

- 無理な強打を控える
- 待っている人を見て交代
- 人が近い場所へのスマッシュを避ける
- 体調が悪い時は無理しない
- 初参加者にも声をかける
- 雰囲気よく楽しむ

セリフは [`lib/flowCautions.ts`](lib/flowCautions.ts) で編集できます。

### 読み上げ

各ステップカードの「▶ 読み上げ」ボタンを押すと、うぃるが進行アナウンスを読み上げます。

- 「読み上げ: VOICEVOX」を選んでいれば **VOICEVOX:ずんだもん** で読み上げ
- VOICEVOX 未接続のときは自動的に **標準音声にフォールバック**
- 「読み上げ: 標準音声」を選んでいればブラウザ標準音声で読み上げ

読み上げ中は「■ 停止」ボタンで止められます。

### コピー・編集

- **コピー**: 画面表示用テキストをそのままクリップボードへ。LINE にそのまま貼れます。
- **編集**: その場でテキストを書き換えできます（簡易テキストエリア）。編集内容は `localStorage` に保存され、編集後は読み上げもコピーも編集後のテキストを使います。「元に戻す」で初期文面に戻せます。

### タイマー

ステップカードの下に5分／10分／カスタム分のタイマーがあります。
タイマー終了時に、うぃるが **「次のメニューに移りましょう」** を自動で読み上げます。

### セリフを編集する（ファイル側）

進行セリフのデータは [`lib/tennisFlowScripts.ts`](lib/tennisFlowScripts.ts) で管理しています。
`FlowScript` 型に従って、`displayText`（画面表示用）と `voiceText`（読み上げ用）の2つを書き分けるだけで自由にセリフを追加・編集できます。

```ts
{
  id: "04-cross-single",
  step: 4,
  title: "クロスラリー (1面)",
  shortLabel: "クロス",
  courtMode: "single", // "single" | "double" | "both"
  displayText: `次はクロスラリーです🎾 …（絵文字あり）`,
  voiceText: `次はクロスラリーです。…（読み上げて自然な文）`,
}
```

`courtMode` を `"single"` / `"double"` に分ければ、コート数で文面を切り替えられます。`"both"` または未指定なら両方で使われます。

## Vercel で公開する（無料枠）

### デプロイ手順

```bash
# ビルドが通ることを確認
npm run build

# GitHub にプッシュ
git add .
git commit -m "Prepare for Vercel deployment"
git push
```

1. このリポジトリを GitHub に push
2. [Vercel](https://vercel.com) で「New Project」→ リポジトリを選択
3. フレームワークは自動で **Next.js** が選ばれます。そのまま Deploy
4. 発行された HTTPS URL をスマホで開けば、マイク・読み上げが動きます🎾

環境変数の設定は不要です（MVPは外部APIを使わないため）。

### 公開版 (Vercel) での音声について

| 機能 | ローカル (`localhost`) | 公開版 (Vercel 等) |
| --- | --- | --- |
| 標準音声 (ブラウザ) | ✅ | ✅ |
| VOICEVOX:ずんだもん | ✅ (VOICEVOX起動時) | ❌ |
| マイク音声認識 | ✅ (Chrome推奨) | ✅ (HTTPS必須) |

**公開版では「標準音声」がメインです。**

Vercel にデプロイすると、Next.js のサーバーは Vercel のクラウドで動作します。VOICEVOX ENGINE はあなたのローカル PC 上で動いているため、**Vercel サーバーからはローカルの `http://127.0.0.1:50021` に接続できません。**

- 公開版で「VOICEVOX」ボタンを選択した場合、パネルに案内メッセージを表示し、読み上げはブラウザ標準音声で行います
- 公開版の初期音声モードは「標準音声」に固定されます（ローカルでの設定はリセットされます）
- 公開版でも VOICEVOX を使いたい場合は、**VOICEVOX ENGINE を VPS 等の別サーバーで起動**し、環境変数 `VOICEVOX_ENGINE_URL` をそのアドレスに設定する構成が必要です

```
# Vercel の環境変数設定例（VPSでVOICEVOX ENGINEを動かす場合）
VOICEVOX_ENGINE_URL=https://your-voicevox-server.example.com
VOICEVOX_DEFAULT_SPEAKER_ID=3
```

## 主要機能と仕組み

| 機能 | 実装 | 場所 |
| --- | --- | --- |
| 音声認識 | Web Speech API (`SpeechRecognition`) | `lib/speech.ts` |
| FAQデータ | 17カテゴリ・120件超のモックルール | `lib/mockReplyRules.ts` |
| 返答生成 | キーワード一致（最長一致を優先）＋フォールバック | `lib/generateWillReply.ts` |
| 読み上げ(標準) | Web Speech API (`SpeechSynthesis`) | `lib/speech.ts` |
| 読み上げ(VOICEVOX) | ローカル VOICEVOX ENGINE → WAV をブラウザで再生 | `lib/voicevox.ts`, `app/api/voicevox/*` |
| キャラ設定 | system prompt として一元管理 | `lib/willPrompt.ts` |
| チャットUI | LINE 風の吹き出し | `app/page.tsx`, `components/` |

### モック返答の仕組み

- FAQは `lib/mockReplyRules.ts` の `MOCK_REPLY_RULES`（`MockReplyRule[]`）で管理します。
- 各ルールは `id` / `category` / `keywords[]` / `reply` を持ちます。言い換えに反応できるよう、1ルールに複数キーワードを登録しています。
- `generateWillReply.ts` は、入力に含まれるキーワードのうち **最も長い（＝具体的な）ものに一致したルール** を選びます。これにより、ルールの並び順に神経質にならなくても自然に動きます。
- どのルールにも当てはまらない場合は、`FALLBACK_REPLY`（うぃるができることの案内）を返します。

#### FAQを増やすには

`lib/mockReplyRules.ts` の配列に、次の形でルールを追加するだけです。

```ts
{
  id: "items-cap",
  category: "持ち物",
  keywords: ["帽子", "キャップ", "日よけ"],
  reply: `お疲れ様です！\n暑い日は帽子があると安心です🎾`,
},
```

## 将来 AI API へ差し替える場所（重要）

コード中に `★将来の差し替えポイント★` というコメントを置いています。

### 1. 返答生成を都度AI（Claude / OpenAI）にする

今はモック返答ですが、**都度AIが適切に回答する構成**にしたい場合は、`generateWillReply(userMessage)` の中身を外部AI API呼び出しに差し替えます。
**関数のシグネチャ `(userMessage: string) => Promise<string>` を保てば、UI 側（`app/page.tsx`）の変更は不要です。**

#### 推奨構成（ブラウザから直接AI APIを呼ばない）

```
[ブラウザ] page.tsx
   └─ generateWillReply(userMessage, history)
        └─ fetch("/api/reply", { userMessage, history })   ← 返答だけ受け取る
                 │
[サーバー] app/api/reply/route.ts
        └─ WILL_SYSTEM_PROMPT + userMessage を AI API に渡す
        └─ APIキーは .env.local（サーバー側だけが知っている）
```

1. **ブラウザから直接AI APIを呼ばない**
2. `app/api/reply/route.ts` を作る（雛形あり → 後述）
3. **APIキーは `.env.local` に保存**する（`.env.example` 参照。`.gitignore` 済み）
4. フロントは `/api/reply` に `userMessage` と会話履歴（`history`）を送る
5. サーバー側で `WILL_SYSTEM_PROMPT` と `userMessage` をAI APIに渡す
6. **返答だけをフロントに返す**

#### この構成にする理由

- **APIキーをブラウザに出さないため**（流出防止。クライアントに置くと誰でも盗める）
- **将来的に利用制限・ログ保存・不正利用対策を入れやすいため**（サーバーが唯一の入口になる）
- **うぃるのキャラ設定（`WILL_SYSTEM_PROMPT`）をサーバー側で固定できるため**（クライアント改変で人格を書き換えられない）

#### サンプルAPIルート

[`app/api/reply/route.ts.example`](app/api/reply/route.ts.example) に雛形を用意しています。
実際に使うときは、このファイルを **`route.ts` にリネーム**し、Claude / OpenAI どちらかのコメントを外して、`.env.local` にキーを入れてください。

- `★ 差し替えポイント A` … Claude API（`@anthropic-ai/sdk`）の例
- `★ 差し替えポイント B` … OpenAI API（`openai`）の例
- AI接続前は `findMockReply()` を返すので、キーが無くても動作確認できます
- AIが落ちたときに `findMockReply()` へフォールバックさせれば、無料モックが保険になります

### 2. 読み上げを「うぃる専用ボイス」にする

`lib/speech.ts` の `speakText()` の中身を TTS API（例: OpenAI TTS）に差し替えます。
取得した音声を `<audio>` で再生する形にすれば、呼び出し側は変更不要です。

### 3. そのほか拡張しやすい設計

- FAQ / よくある質問 → `lib/mockReplyRules.ts` の `MOCK_REPLY_RULES` に追記
- 管理者向け案内文作成モード / 参加者向けQ&Aモード → モード引数を追加
- LINE Bot 連携・日程情報連携 → サーバー側 Route Handler を追加

## うぃるの口調について

うぃるの人格・口調・絵文字ルール・NG表現は `lib/willPrompt.ts` に
system prompt としてまとめてあります。AI API に切り替えても、ここを渡せば
同じ口調を保てます。

> お疲れ様です！うぃるです🎾 これからもよろしくお願いいたします！
